# 约束金字塔

Archon 的质量保障不靠文档——文档是最弱的约束。每条值得遵守的规则都往**约束力最强的层级推**。

## 六级约束

```
        ╔═══════════════════════════════════════╗
        ║  L0  类型系统                          ║  编译不过 = 不存在
        ╠═══════════════════════════════════════╣
        ║  L1  Linter  ←── Lint-Rule 桥接 ──→  ║  违规 = 不提交
        ╠═══════════════════════════════════════╣
        ║  L2  编辑器规则                        ║  编辑时提示
        ╠═══════════════════════════════════════╣
        ║  L3  技能文档                          ║  按需索引
        ╠═══════════════════════════════════════╣
        ║  L4  架构决策记录 (ADR)                ║  历史溯源
        ╠═══════════════════════════════════════╣
        ║  L5  Manifest                         ║  上下文同步
        ╚═══════════════════════════════════════╝
         ▲ 约束力最强                 约束力最弱 ▼
```

**金律：能让机器拒绝的不写文字提醒。文档只承载"为什么"和机器无法表达的设计意图。**

## 各层级详解

### L0 — 类型系统

**强度：编译不过 = 不存在。**

能被类型约束的接口契约、数据形状，全部用类型表达。TypeScript 的 `strict` 模式、Rust 的所有权系统、Python 的 `mypy --strict`——用你的语言提供的最严格模式。

### L1 — Linter

**强度：违规 = 不提交。**

编码约定、架构边界、命名规范。Linter 的优势是**主动推送**——AI 不需要记住规则，违规时错误信息直接告诉它怎么修。

### L2 — 编辑器规则

**强度：编辑时提示。**

AI 编程平台的规则文件（`.cursor/rules/`、`.claude/` 等）。跨文件的编码模式、设计系统约束。

**弱点**：被动——AI 没加载这个文件就等于不存在。

### L3 — 技能文档

**强度：按需索引。**

可复用的领域知识：设计体系、API 规范、框架最佳实践。在 AI 编程平台的技能目录下。

### L4 — 架构决策记录

**强度：历史溯源。**

有背景、对比、取舍的架构选择。每条包含状态（生效 / 已废弃 / 否决）、理由和影响。

### L5 — Manifest

**强度：上下文同步。**

项目状态、技术栈、目录结构变更。Archon 每次启动时加载。

## 约束成熟度

每条约束都有一个朝更强执行力方向的升级路径：

```
发现 ──→ SHOULD (写入文档/skill) ──→ MUST (变成 lint 规则或测试)
```

**停留在文档层而本可以机械化验证的约束 = 可靠性缺口。**

实践中的判断标准：
- 这条约束能被正则匹配检测吗？→ 推到 L1 (lint)
- 能被类型系统表达吗？→ 推到 L0
- 两者都不行？→ 留在 L2/L3，但写清楚**为什么**机器做不了

## Lint-Rule 桥接

L2 编辑器规则的致命弱点是被动性——AI 没读到就等于不存在。解法：

```
代码违规 → Linter 报错
         → 错误信息包含：→ Read .cursor/rules/xxx.mdc
         → AI 读到路径 → 主动加载完整规范 → 按规范修复
```

**效果**：每次 lint 违规 = 一次规范的**强制推送**。L1 主动触发 L2。

编写约定：错误信息 = 自包含修复指引（前半段，不读文件也能改）+ `→ Read <path>`（后半段，完整规范）。

```javascript
// ESLint 示例
{
  message: 'Pages should import from hooks, not @/lib/api directly. '
         + '→ Read .cursor/rules/frontend.mdc',
}
```

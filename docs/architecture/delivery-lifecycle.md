# 交付生命周期

每次 `/archon-demand` 触发的完整流程：

```
启动 ─→ 前置扫描 ─→ 决策关卡 ─→ 执行 ─→ 验证门 ─→ 收尾 ─→ 版本控制
           │                                        │
    检索对话备忘                           ┌─────────┴─────────┐
    + 架构决策                            │   6 步收尾流程       │
    命中→引用历史                          │                     │
                                          │ ① manifest 同步     │
                                          │ ② 知识捕获(sub)     │
                                          │ ③ 沉淀执行           │
                                          │ ④ drift 评分+记录    │
                                          │ ⑤ 里程碑门禁         │
                                          │ ⑥ 对话备忘更新       │
                                          └─────────────────────┘
```

## 决策关卡

**动手前**必须回答三个问题：

| 问题 | 判断标准 |
|------|---------|
| **该不该做** | 解决的是真实问题还是假设问题？有无不改代码的解法？ |
| **做多大** | 爆炸半径（影响多少文件）+ 可逆性（回滚成本） |
| **谁来定** | 工程决策自己定；产品方向问用户 |

评估为"不做"时——**否决需求并给出理由和替代方案**。这是 Agent 的职责，不是越权。

## 前置扫描

收到需求后，Agent 在动手前先检索两个信息源：

1. **对话备忘表**（manifest 内）：之前讨论过这个话题吗？有没有已做的决策？
2. **架构决策记录**（ADR）：有没有相关的架构约束？

如果命中，主动引用历史结论。这保证了跨会话的决策一致性——用户不需要重复自己。

## 验证门

交付后必须执行 manifest 中声明的验证命令。

**红了修到绿，不存在"先过了回头再说"。**

## 收尾六步

| 步骤 | 执行者 | 做什么 |
|------|--------|--------|
| ① Manifest 同步 | 主 Agent | 更新目录结构、知识资产索引、里程碑进度 |
| ② 知识捕获 + 交付卫生 | **capture-auditor** (sub-agent) | 评估是否产生了值得固化的经验 + 检查交付卫生 |
| ③ 沉淀执行 + 卫生响应 | 主 Agent | 按 auditor 建议执行沉淀 / 修复卫生问题 |
| ④ Drift 评分 + 记录 | 主 Agent | 评估复杂度，累加漂移分，写入日志 |
| ⑤ 里程碑门禁 | 主 Agent | 检查里程碑验收条件是否全部达成 |
| ⑥ 对话备忘更新 | 主 Agent | 有决策价值的结论写入备忘表 |

步骤 ② 是核心创新——交付者对自己的工作有沉没成本偏差，所以知识捕获评估委托给独立的 sub-agent。详见 [Sub-agent 委托](/architecture/sub-agent)。

## 阈值事件

当 drift 累计 ≥ 12 时，下次交付前**强制执行全量审查**（`/archon-review`）。审查由独立的 reviewer sub-agent 完成，主 Agent 根据审查报告逐项修复。

审查后 drift 重置：
- 完全清晰 → 归 0
- 有残留疑点 → 写入 1-3，附说明

## 版本控制

Archon 支持三种 Git 模式（在 manifest 中声明）：

| 模式 | 行为 |
|------|------|
| `auto` | 交付后自动 commit |
| `prompt` | 交付后确认是否 commit |
| `off` | 用户自行处理 |

分支模型同样在 manifest 中声明（`direct` / `feature-branch` / `trunk-based`）。

## 路由

从输入中确定工作模式。两种方式，显式优先：

### 显式指定（省 token）

输入以下列前缀开头时直接路由，前缀后的文本作为目标模式的输入：

| 前缀 | 目标 |
|------|------|
| `plan` | 规划 |
| `review` | 审查 |
| `demand:` | 交付 |

### 隐式识别

无匹配前缀时，从语义判断：

| 意图信号 | 目标 |
|----------|------|
| 问状态、问方向、问下一步、不确定做什么 | plan |
| 要求审查、检查质量、查隐患 | review |
| 具体需求、bug、功能、变更 | demand |
| 空输入 / 打招呼 | plan |

不明确时偏向 plan——只读安全、信息丰富。

## 执行

1. 阅读同级命令目录下对应的命令文件（archon-plan.md / archon-review.md / archon-demand.md）
2. soul.md 和 manifest.md 已在上下文中——跳过命令文件首行的加载指令
3. 将路由后的剩余文本作为该模式的 `{{input}}`
4. 严格按命令文件中的流程执行，不增删步骤

路由到 demand 时，先阅读 drift.md。drift ≥ 阈值 → 必须先完成全量审查（archon-review 流程），再回来执行 demand。
